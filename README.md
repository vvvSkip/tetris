# 俄罗斯方块小游戏使用说明

## 项目简介

这是一个基于 uni-app、Vue 3、TypeScript 和 SCSS 开发的移动端俄罗斯方块小游戏，支持 H5 和微信小程序运行环境。页面已按移动端优先设计，棋盘、信息面板和触控按钮会在常见手机屏幕内完成展示，避免按钮区域被挤到首屏之外。

## 功能特性

- 经典 10 x 20 俄罗斯方块棋盘。
- 支持开始、暂停、继续、重开。
- 支持左移、右移、旋转、下落、直落。
- 支持下一块预览。
- 支持落点幽灵提示，方便判断方块最终位置。
- 支持分数、消除行数、等级统计。
- 支持每日免费游玩次数、积分奖励和积分兑换使用次数。
- 支持兑换记录页面，查看每次积分兑换的积分变化和兑换后次数。
- 支持防沉迷健康提醒，累计游玩达到设定时长后提示休息。
- 支持游戏记录页面，查看最近 50 局成绩和每局获得积分。
- 支持在主页面打开规则说明弹窗，快速查看玩法、计分、积分和健康提醒规则。
- 支持浅色和深色主题切换，主题选择会保存在本地。
- 支持游戏音效开关，移动、旋转、下落、消行和游戏结束等操作会有轻量音效反馈。
- 随等级提升自动加快下落速度。
- H5 支持键盘操作，移动端和微信小程序支持触控按钮操作。
- 使用 `nutui-uniapp` 的 `nut-button` 作为基础按钮组件。

## 技术栈

- uni-app
- Vue 3
- TypeScript
- Vite
- SCSS
- NutUI UniApp

## 环境要求

建议使用以下环境：

- Node.js 18 或更高版本
- npm 9 或更高版本
- 微信开发者工具，用于预览和调试微信小程序

首次运行前请确认已经安装依赖：

```bash
npm install
```

## 快速开始

### H5 本地运行

```bash
npm run dev
```

命令启动后，终端会输出本地访问地址，通常类似：

```text
http://localhost:5173/
```

在浏览器中打开该地址即可体验 H5 版本。

### H5 构建

```bash
npm run build
```

构建产物会输出到 `dist/build/h5` 目录，可用于部署到静态服务器。

### 微信小程序本地运行

```bash
npm run dev:mp-weixin
```

运行后使用微信开发者工具导入生成的小程序目录进行预览。通常目录位于：

```text
dist/dev/mp-weixin
```

### 微信小程序构建

```bash
npm run build:mp-weixin
```

构建产物通常位于：

```text
dist/build/mp-weixin
```

可在微信开发者工具中导入该目录后进行上传或体验版调试。

## 游戏操作说明

### 触控按钮

页面底部提供移动端触控按钮：

| 按钮 | 作用 |
| --- | --- |
| 左 | 当前方块向左移动一格 |
| 旋转 | 当前方块顺时针旋转 |
| 右 | 当前方块向右移动一格 |
| 下落 | 当前方块向下移动一格，成功下落会增加少量分数 |
| 开始 / 暂停 / 继续 | 控制游戏状态 |
| 直落 | 当前方块直接落到底部，并按下落距离加分 |
| 重开 | 重置棋盘、分数、行数和等级 |

当游戏处于准备、暂停或结束状态时，移动、旋转、下落和直落按钮会按当前状态禁用，避免误操作。

### 更多菜单

首页右上角提供“更多”入口，可打开以下功能：

| 功能 | 说明 |
| --- | --- |
| 规则说明 | 以弹窗方式查看玩法、计分、积分、次数和健康提醒规则 |
| 游戏记录 | 进入游戏记录页面，查看最近 50 局结算 |
| 兑换记录 | 进入兑换记录页面，查看积分兑换明细 |
| 切换主题 | 在浅色主题和深色主题之间切换，切换后会自动保存 |
| 音效开关 | 控制游戏音效开启或关闭，设置会自动保存 |

### H5 键盘操作

H5 环境额外支持键盘操作：

| 按键 | 作用 |
| --- | --- |
| 方向键左 | 左移 |
| 方向键右 | 右移 |
| 方向键上 | 旋转 |
| 方向键下 | 下落 |
| 空格 | 直落 |
| Enter | 开始、暂停或继续 |

## 计分规则

当前计分规则如下：

| 行为 | 分数 |
| --- | --- |
| 手动下落一格 | +1 |
| 直落 | 下落距离 x 2 |
| 一次消除 1 行 | 100 x 当前等级 |
| 一次消除 2 行 | 300 x 当前等级 |
| 一次消除 3 行 | 500 x 当前等级 |
| 一次消除 4 行 | 800 x 当前等级 |

每累计消除 10 行提升 1 级。等级越高，方块自动下落间隔越短，游戏速度越快。

## 结算评分和游戏记录规则

每局游戏结束时会自动进行结算，并保存到“游戏记录”页面。结算分由游戏内基础得分、时长奖励和等级奖励组成。

```text
最终得分 = 游戏内基础得分 + 时长奖励 + 等级奖励
```

| 项目 | 规则 |
| --- | --- |
| 游戏内基础得分 | 游戏过程中通过下落、直落和消行获得的分数 |
| 时长奖励 | 每实际游玩 10 秒 +10 分，最高 +600 分 |
| 等级奖励 | 从 2 级开始，每提升 1 级 +150 分 |
| 实际游玩时长 | 只统计游戏中状态的时间，暂停时间不计入 |

评级规则如下：

| 评级 | 达成条件 |
| --- | --- |
| S | 最终得分达到 5000，或累计消除 40 行 |
| A | 最终得分达到 3000，或累计消除 25 行 |
| B | 最终得分达到 1500，或累计消除 12 行 |
| C | 最终得分达到 500，或累计消除 4 行 |
| D | 未达到以上条件 |

游戏记录规则：

- 每局结束后自动保存一条记录。
- 最多保留最近 50 局记录。
- 记录内容包含最终得分、评级、基础分、时长奖励、等级奖励、获得积分、消行数、等级、用时和结束时间。
- 在“游戏记录”页面可以查看总局数、最高分、最佳评级和累计消行。
- 清空记录只删除历史记录，不会清空当前积分和兑换次数。

## 积分和次数规则

当前积分规则如下：

| 规则 | 说明 |
| --- | --- |
| 每日免费次数 | 每天免费 5 次，按自然日自动刷新 |
| 开始新局 | 从准备或结束状态开始游戏会消耗 1 次 |
| 暂停 / 继续 | 不消耗次数 |
| 兑换次数 | 100 积分可兑换 1 次额外游戏次数 |
| 积分奖励 | 每局结束后按最终得分发放积分 |

积分奖励计算方式：

```text
基础积分 = 最终得分 / 100 后向下取整
评级额外奖励：S +30，A +20，B +10，C +5，D +0
最终获得积分 = 基础积分 + 评级额外奖励
```

当今日免费次数和兑换次数都用完时，无法开始新局，需要先使用积分兑换次数。

兑换记录规则：

- 每次积分兑换成功后自动保存一条兑换记录。
- 记录内容包含兑换时间、消耗积分、兑换前积分、兑换后积分、兑换次数和兑换后的剩余兑换次数。
- 在“兑换记录”页面可以查看全部兑换记录。
- 暂无兑换记录时会展示空状态布局。

## 防沉迷提醒规则

当前防沉迷机制只做提醒，不会强制退出或禁止继续游戏。

| 规则 | 说明 |
| --- | --- |
| 统计范围 | 按自然日统计当天累计实际游玩时长 |
| 首次提醒 | 当天累计游玩达到 30 分钟 |
| 后续提醒 | 首次提醒后，每累计增加 15 分钟再次提醒 |
| 暂停状态 | 暂停、结束、离开页面时停止计时 |
| 提醒方式 | 弹出健康提醒弹窗，提示玩家休息 |

提醒只用于健康提示，不影响积分、次数、游戏记录和当前游戏进度。

## 音效规则

音效默认为开启，可在首页右上角“更多”菜单中关闭或重新开启。开关状态会保存在本地，下次进入游戏时自动沿用。

当前音效反馈如下：

| 场景 | 说明 |
| --- | --- |
| 开始 / 继续 | 播放短促上扬提示音 |
| 暂停 | 播放低频提示音 |
| 左移 / 右移 | 方块成功移动时播放轻提示音 |
| 旋转 | 方块成功旋转时播放轻提示音 |
| 下落 / 直落 | 方块成功下落或直落时播放下落提示音 |
| 消行 | 成功消除行时播放连续奖励音 |
| 游戏结束 | 播放结束提示音 |

H5 环境使用浏览器 Web Audio 生成轻量音效；微信小程序环境优先使用可用的 WebAudio 能力，不支持时保留触感反馈作为兜底。

## 页面说明

主页面由多个页面级组件、游戏逻辑组合函数和积分组合函数组成：

```text
src/pages/index/index.vue
src/pages/index/components/creditPanel.vue
src/pages/index/components/gameBoard.vue
src/pages/index/components/gamePanel.vue
src/pages/index/components/touchControls.vue
src/pages/index/use/useTetris.ts
src/hooks/useAntiAddiction.ts
src/hooks/useGameCredits.ts
src/hooks/useGameRecords.ts
src/hooks/useSoundEffects.ts
```

### `index.vue`

页面组合层，负责：

- 组织信息面板、棋盘和触控按钮。
- 调用 `useTetris()` 获取游戏状态和操作方法。
- 调用 `useGameCredits()` 处理次数消耗、积分发放和兑换。
- 调用 `useAntiAddiction()` 处理每日累计游玩时长和健康提醒。
- 调用 `useSoundEffects()` 处理音效开关和游戏操作音效。
- 在 H5 环境注册键盘事件。
- 设置页面高度、背景、安全区和首屏布局。

### `creditPanel.vue`

积分和次数面板组件，负责：

- 展示当前积分。
- 展示今日剩余免费次数和可玩次数。
- 提供积分兑换次数入口。
- 提供兑换记录页面入口。
- 提供规则说明弹窗入口。

### `rulePopup.vue`

规则说明弹窗组件，负责：

- 通过主页面入口展示玩法、计分、积分、次数、记录和健康提醒规则。
- 使用 `nut-popup` 从底部弹出，并适配移动端安全区。
- 通过 `changeShow()` 方法供主页面控制显隐。

### `gameBoard.vue`

棋盘展示组件，负责：

- 渲染 10 x 20 游戏格子。
- 根据方块类型展示不同颜色。
- 展示幽灵落点。
- 针对小高度屏幕使用更紧凑的棋盘尺寸。

### `gamePanel.vue`

信息面板组件，负责：

- 展示分数、行数、等级。
- 展示当前游戏状态。
- 展示下一块方块预览。

### `touchControls.vue`

触控操作组件，负责：

- 渲染移动端操作按钮。
- 使用 `nut-button` 保持按钮体验一致。
- 通过事件向父组件传递用户操作。

### `useTetris.ts`

核心游戏逻辑，负责：

- 棋盘初始化和重置。
- 方块生成、移动、旋转和碰撞检测。
- 自动下落计时器。
- 方块锁定和消行。
- 分数、行数、等级、游戏状态计算。
- 下一块预览和幽灵落点计算。
- 游戏结束后触发结算回调。

### `useGameCredits.ts`

积分和次数逻辑，负责：

- 管理本地积分余额。
- 管理每日免费次数和兑换次数。
- 处理开始新局时的次数消耗。
- 处理积分兑换次数。
- 保存和读取兑换记录。
- 计算每局结束后的积分奖励。

### `useAntiAddiction.ts`

防沉迷提醒逻辑，负责：

- 按自然日记录累计实际游玩时长。
- 在开始、暂停、结束、离开页面时维护计时状态。
- 在累计游玩达到提醒阈值时触发健康提醒。
- 只做提醒，不限制继续游戏。

### `useSoundEffects.ts`

音效逻辑，负责：

- 管理音效开关状态。
- 使用本地存储保存用户偏好。
- 为移动、旋转、下落、消行、开始、暂停和结束等场景提供音效播放方法。
- 在 H5 和微信小程序中分别使用兼容的播放能力。

### `useGameRecords.ts`

游戏记录逻辑，负责：

- 保存最近 50 局游戏记录。
- 计算最终得分、评级和每局获得积分。
- 为游戏记录页面提供汇总数据。

## 移动端适配说明

当前页面采用移动端优先布局：

- 根页面使用固定视口高度，并隐藏页面滚动，避免操作区被挤到屏幕外。
- 页面使用自定义导航配置隐藏原生标题栏，将更多首屏空间留给棋盘和操作区。
- 主体内容使用纵向 flex 布局，通过 `justify-content: space-between` 分配积分面板、信息面板、棋盘和按钮区域。
- 尺寸优先使用 `rpx`，便于不同屏幕宽度下按比例缩放。
- 页面底部使用 `env(safe-area-inset-bottom)` 处理安全区域，适配全面屏手机。
- 棋盘在低高度屏幕下会进入紧凑尺寸，减少页面溢出风险。

建议重点验证以下视口：

- 375 x 667，小屏手机。
- 390 x 844，常见全面屏手机。
- 414 x 896，大屏手机。
- 微信开发者工具中的 iPhone 机型模拟器。

## 跨端兼容说明

项目目标运行环境为 H5 和微信小程序：

- 页面结构使用 uni-app 组件，如 `view`、`text`。
- H5 键盘事件使用条件编译包裹，只在 H5 环境启用。
- 组件中设置了 `styleIsolation: 'shared'`，便于 NutUI UniApp 样式在微信小程序中正确覆盖。
- NutUI UniApp 通过 `pages.json` 的 `easycom` 自动导入，不需要在每个组件中手动注册。

## 常见问题

### 1. 页面出现滚动条或按钮不在首屏内

请确认当前代码中的根页面和主页面高度未被改动：

- `src/App.vue` 应保持根节点占满视口并隐藏溢出。
- `src/pages/index/index.vue` 的 `.tetris-page` 和 `.tetris-page__content` 应保持固定高度和 `overflow: hidden`。
- 不建议在主页面外层新增会撑高页面的内容。

### 2. 微信小程序中 NutUI 按钮样式异常

请确认组件中保留：

```ts
defineOptions({
  options: {
    styleIsolation: 'shared'
  }
})
```

同时确认 `src/pages.json` 中存在 `easycom` 配置：

```json
{
  "easycom": {
    "autoscan": true,
    "custom": {
      "^nut-(.*)": "nutui-uniapp/components/$1/$1.vue"
    }
  }
}
```

### 3. 构建时出现 Sass `@import` 废弃提示

如果提示来自 NutUI UniApp 内部样式依赖，通常不影响运行和构建结果。只要命令最终成功退出即可。

### 4. H5 键盘操作没有响应

请先点击游戏页面，让浏览器焦点回到当前页面，再使用方向键、空格或 Enter 操作。部分浏览器插件也可能拦截快捷键。

### 5. 微信开发者工具无法打开小程序目录

请先运行：

```bash
npm run dev:mp-weixin
```

然后在微信开发者工具中导入 `dist/dev/mp-weixin`，不要直接导入项目根目录。

## 开发注意事项

- 不要运行格式化或 lint 命令，避免与项目规范冲突。
- 页面组件继续放在 `src/pages/index/components/` 下。
- 当前页面相关逻辑继续放在 `src/pages/index/use/` 下。
- 新增基础按钮、反馈、弹窗等 UI 时，优先使用 `nutui-uniapp`。
- 修改移动端布局后，至少验证一个小屏和一个常见全面屏尺寸。
- 修改游戏核心逻辑后，建议同时验证 H5 构建和微信小程序构建。
